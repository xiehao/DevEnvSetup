# macOS 手动安装指南

> **适用场景**：当自动化脚本 `Install.sh` 失效或中断时，可按本指南手动完成环境配置。

---

## 📋 前置准备

### 系统要求
- **操作系统**：macOS 13 (Ventura) 或更高版本
- **硬件要求**：
  - 剩余磁盘空间 > 20GB
  - Apple Silicon (M1/M2/M3/M4) 或 Intel Mac 均可
- **网络**：首次安装需联网（Homebrew 及 Podman 初始化）

### 安装 Xcode 命令行工具

Homebrew 依赖 Xcode 命令行工具，请先确认已安装：

```bash
xcode-select -p
```

如果提示未安装，执行：

```bash
xcode-select --install
```

在弹出窗口中点击"安装"，等待完成后继续。

## 第一步：安装 Homebrew（国内镜像）

### 1.1 设置镜像环境变量

```bash
export HOMEBREW_BREW_GIT_REMOTE="https://mirrors.tuna.tsinghua.edu.cn/git/homebrew/brew.git"
export HOMEBREW_CORE_GIT_REMOTE="https://mirrors.tuna.tsinghua.edu.cn/git/homebrew/homebrew-core.git"
export HOMEBREW_INSTALL_FROM_API=1
export HOMEBREW_API_DOMAIN="https://mirrors.tuna.tsinghua.edu.cn/homebrew-bottles/api"
export HOMEBREW_BOTTLE_DOMAIN="https://mirrors.tuna.tsinghua.edu.cn/homebrew-bottles"
```

### 1.2 执行安装

```bash
git clone --depth=1 https://mirrors.tuna.tsinghua.edu.cn/git/homebrew/install.git brew-install
/bin/bash brew-install/install.sh
rm -rf brew-install
```

### 1.3 配置 Shell 环境（关键步骤）

Apple Silicon Mac 上 Homebrew 安装在 `/opt/homebrew`，**必须执行以下命令才能使 `brew` 生效**：

```bash
eval "$(/opt/homebrew/bin/brew shellenv)"
```

> Intel Mac 上 Homebrew 安装在 `/usr/local`，通常无需额外操作。如果 `brew` 命令仍找不到，执行 `eval "$(/usr/local/bin/brew shellenv)"`。

### 1.4 持久化环境变量

将以下内容写入 `~/.zprofile`，确保重启终端后依然生效：

```bash
cat >> ~/.zprofile << 'EOF'

# Homebrew mirrors for CN
export HOMEBREW_BREW_GIT_REMOTE="https://mirrors.tuna.tsinghua.edu.cn/git/homebrew/brew.git"
export HOMEBREW_CORE_GIT_REMOTE="https://mirrors.tuna.tsinghua.edu.cn/git/homebrew/homebrew-core.git"
export HOMEBREW_INSTALL_FROM_API=1
export HOMEBREW_API_DOMAIN="https://mirrors.tuna.tsinghua.edu.cn/homebrew-bottles/api"
export HOMEBREW_BOTTLE_DOMAIN="https://mirrors.tuna.tsinghua.edu.cn/homebrew-bottles"
eval "$(/opt/homebrew/bin/brew shellenv)"
EOF
```

### 1.5 切换 Homebrew 仓库远程地址

确保后续 `brew update` 也走国内镜像：

```bash
git -C "$(brew --repo)" remote set-url origin "https://mirrors.tuna.tsinghua.edu.cn/git/homebrew/brew.git"
git -C "$(brew --repo homebrew/core)" remote set-url origin "https://mirrors.tuna.tsinghua.edu.cn/git/homebrew/homebrew-core.git"
```

### 1.6 验证 Homebrew 安装

```bash
brew update
brew --version
```

## 第二步：安装 Podman 与 VSCode

```bash
brew install podman
brew install --cask visual-studio-code
```

安装完成后验证：

```bash
podman --version
code --version
```

## 第三步：初始化 Podman 虚拟机

### 3.1 使用本地镜像初始化（推荐，离线可用）

如果项目根目录的 `Resources` 中存在 **applehv 格式**的镜像文件（如 `podman-machine-aarch64.applehv.raw.gz`）：

```bash
rootfs=$(ls ../Resources/podman-machine-*.applehv.* 2>/dev/null | head -n 1)
podman machine init --image-path "$rootfs" --rootful
```

> **注意**：`Resources` 中可能存在 `podman-machine-*-wsl.tar.zst` 文件，这是 **Windows WSL 专用镜像**，macOS 上**不可使用**。macOS 使用 Apple Hypervisor Framework，需要 applehv 格式的镜像。
>
> 如果没有 applehv 镜像，请先运行 `bash DownloadResources.sh` 下载，或直接使用网络初始化。

### 3.2 通过网络初始化（需要联网）

```bash
podman machine init --rootful
```

`--rootful` 参数表示以 root 权限运行容器，这对于某些开发场景是必需的。

## 第四步：配置国内镜像加速（推荐）

由于国内网络环境，建议为 Docker Hub 配置国内镜像源：

```bash
cat <<'EOF' > /tmp/99-cn-mirrors.conf
unqualified-search-registries = ["docker.io"]

[[registry]]
prefix = "docker.io"
location = "docker.io"

[[registry.mirror]]
location = "docker.m.daocloud.io"

[[registry.mirror]]
location = "mirror.ccs.tencentyun.com"
EOF

podman machine start
podman machine ssh -- "sudo mkdir -p /etc/containers/registries.conf.d"
podman machine ssh -- "sudo tee /etc/containers/registries.conf.d/99-cn-mirrors.conf > /dev/null" < /tmp/99-cn-mirrors.conf
podman machine stop
```

## 第五步：导入或构建开发容器镜像

### 5.1 启动虚拟机

```bash
podman machine start
```

### 5.2 导入或构建镜像

#### 方法 A：从本地压缩包导入（最快）

如果 `Resources` 中存在 `arch-cpp-dev.tar.zst`：

```bash
podman load -i "../Resources/arch-cpp-dev.tar.zst"
```

#### 方法 B：从 Dockerfile 构建（需要联网）

如果 `Resources` 中存在 `Dockerfile`：

```bash
# 构建镜像（需要 10-30 分钟，取决于网络速度）
podman build -t arch-cpp-dev:latest -f "../Resources/Dockerfile" "../Resources"
```

### 5.3 验证镜像导入成功

```bash
podman images
# 应该能看到 arch-cpp-dev 镜像
```

### 5.4 测试运行容器

```bash
# 启动容器并进入交互式 Shell
podman run -it --rm arch-cpp-dev:latest

# 在容器内测试编译器
gcc --version
clang++ --version

# 退出容器
exit
```

## 第六步：配置 VSCode 开发环境

### 6.1 导入 VSCode 配置文件

1. 打开 **VSCode**
2. 点击左下角的 **齿轮图标 (⚙️)** → 选择 **"配置文件 (Profiles)"**
3. 点击 **"导入配置文件... (Import Profile...)"**
4. 在文件选择对话框中，导航到项目根目录：
   ```
   DevEnvSetup/Resources/Container.code-profile
   ```
5. 选择该文件并确认导入

### 6.2 配置文件包含的内容

导入的配置文件预配置了以下内容：
- **扩展插件**：C/C++, CMake Tools, Dev Containers, CodeLLDB 等
- **容器连接设置**：自动识别 Podman 容器
- **编辑器设置**：代码格式化、智能提示等
- **调试配置**：GDB/LLDB 调试器配置

### 6.3 连接到容器

1. 在 VSCode 中按 `F1`（或 `Cmd+Shift+P`）打开命令面板
2. 输入并选择：`Dev Containers: Attach to Running Container...`
3. 选择 `arch-cpp-dev` 容器
4. VSCode 会在新窗口中打开容器内的开发环境

### 6.4 （可选）手动安装必要扩展

如果配置文件导入失败，可手动安装以下扩展：

```
必需扩展：
- ms-vscode.cpptools (C/C++)
- ms-vscode.cmake-tools (CMake Tools)
- ms-vscode-remote.remote-containers (Dev Containers)
- vadimcn.vscode-lldb (CodeLLDB)

推荐扩展：
- ms-vscode.cpptools-extension-pack (C/C++ Extension Pack)
- twxs.cmake (CMake Language Support)
```

## 🔧 故障排查

### 问题 1：Homebrew 安装失败或速度慢

**症状**：`git clone` 卡住或报网络错误

**解决方案**：
1. 确认网络连接正常，可尝试 `ping mirrors.tuna.tsinghua.edu.cn`
2. 如果使用代理，设置终端代理：
   ```bash
   export HTTP_PROXY="http://127.0.0.1:7890"
   export HTTPS_PROXY="http://127.0.0.1:7890"
   ```
3. 尝试使用官方安装脚本（需要直接访问 GitHub）：
   ```bash
   /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
   ```

### 问题 2：`brew` 命令找不到

**症状**：安装完成后执行 `brew` 提示 `command not found`

**解决方案**：
1. Apple Silicon Mac：
   ```bash
   eval "$(/opt/homebrew/bin/brew shellenv)"
   ```
2. Intel Mac：
   ```bash
   eval "$(/usr/local/bin/brew shellenv)"
   ```
3. 确认已写入 `~/.zprofile`，然后重启终端或执行 `source ~/.zprofile`

### 问题 3：Podman 虚拟机启动失败

**症状**：执行 `podman machine start` 时报错

**解决方案**：
1. 确认 macOS 版本不低于 13 (Ventura)
2. 重置 Podman 虚拟机：
   ```bash
   podman machine stop 2>/dev/null
   podman machine rm -f
   podman machine init --rootful
   podman machine start
   ```
3. 如果提示虚拟化相关错误，检查"系统设置" → "隐私与安全性"中是否有被阻止的系统扩展

### 问题 4：容器镜像构建失败

**症状**：执行 `podman build` 时报网络错误或包下载失败

**解决方案**：
1. 确保已配置国内镜像加速（第四步）
2. 尝试更换 Arch Linux 镜像源（编辑 `../Resources/Dockerfile`）：
   ```dockerfile
   RUN echo "Server = https://mirrors.ustc.edu.cn/archlinux/\$repo/os/\$arch" > /etc/pacman.d/mirrorlist
   ```
3. 分步构建，逐行执行 Dockerfile 中的命令以定位问题

### 问题 5：VSCode 无法连接到容器

**症状**：VSCode 提示"无法连接到容器"

**解决方案**：
1. 确认容器正在运行：
   ```bash
   podman ps -a
   ```
2. 手动启动一个长期运行的容器：
   ```bash
   podman run -d --name dev-container arch-cpp-dev:latest sleep infinity
   ```
3. 确保已安装 "Dev Containers" 扩展并重启 VSCode
4. 检查 Podman 是否在 PATH 中：
   ```bash
   which podman
   ```

### 问题 6：磁盘空间不足

**症状**：安装过程中提示磁盘空间不足

**解决方案**：
1. 清理 Homebrew 缓存：
   ```bash
   brew cleanup --prune=all
   ```
2. 清理 Podman 虚拟机和镜像：
   ```bash
   podman system prune -a
   ```
3. 检查虚拟机磁盘占用：
   ```bash
   podman machine inspect --format "{{.Resources.DiskSize}}"
   ```

## ✅ 安装完成验证

执行以下命令验证环境是否配置成功：

```bash
# 1. 检查 Homebrew
brew --version

# 2. 检查 Podman
podman --version

# 3. 检查 VSCode
code --version

# 4. 检查 Podman 虚拟机状态
podman machine list

# 5. 检查容器镜像
podman images

# 6. 运行测试容器
podman run -it --rm arch-cpp-dev:latest gcc --version
```

如果以上命令均正常执行，说明环境配置成功！
